# Bazar — старт проекта (Модуль 0)

Это стартовый каркас монорепо: Next.js-приложение, подключённое к Supabase,
и первая SQL-миграция (роли, профили, география, категории, настройки сайта).
Полный дизайн, объявления, чат, монетизация и остальное — по roadmap, модуль
за модулем.

## Что уже готово в этом архиве

- `apps/web` — Next.js 15 (App Router) + TypeScript + Tailwind, подключение к Supabase (SSR и браузер)
- `supabase/migrations/0001_init.sql` — таблицы `roles`, `profiles`, `user_roles`, `regions`, `cities`, `categories`, `category_attributes`, `site_settings` + RLS-политики
- `supabase/seed.sql` — районы/города Ингушетии и 19 базовых категорий из ТЗ
- Главная страница читает название сайта и категории напрямую из базы — это уже проверяет, что вся цепочка Frontend → Supabase работает

## Шаг 1. Создать проект в Supabase

1. Зайдите на **supabase.com** → **Start your project** → войдите через GitHub (это бесплатно на старте, тариф Free).
2. **New project** → укажите:
   - Name: `ing-market`
   - Database password: сгенерируйте и сохраните в надёжном месте (понадобится для прямых подключений/бэкапов)
   - Region: выбирайте ближайший к пользователям регион (обычно Frankfurt/Europe для РФ-аудитории — минимальный пинг)
3. Дождитесь создания проекта (1–2 минуты).
4. В левом меню откройте **Project Settings → API** и скопируйте:
   - `Project URL` → это `NEXT_PUBLIC_SUPABASE_URL`
   - `anon public` ключ → это `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `service_role` ключ → это `SUPABASE_SERVICE_ROLE_KEY` (никогда не публикуется, только на сервере)

### Применить миграцию и seed-данные

Проще всего через **SQL Editor** в панели Supabase:

1. Откройте **SQL Editor** → **New query**.
2. Вставьте содержимое `supabase/migrations/0001_init.sql`, нажмите **Run**.
3. Затем вставьте содержимое `supabase/seed.sql`, нажмите **Run**.

(Позже, когда появятся следующие миграции, удобнее перейти на Supabase CLI —
`supabase db push` — это будет описано в README к Модулю 1.)

### Включить провайдеры входа

**Authentication → Providers**:
- Email — включён по умолчанию, в **Email Templates** можно сразу поправить письма на русский.
- Phone — потребуется подключить SMS-провайдера (SMS.ru/SMSC) в Модуле 1.
- Google — **Authentication → Providers → Google**, потребуется Client ID/Secret из Google Cloud Console (сделаем в Модуле 1 вместе с формой регистрации).

## Шаг 2. Локальный запуск

Понадобится **Node.js 20+** и **pnpm** (`npm install -g pnpm`).

```bash
cd ing-market
pnpm install

cp apps/web/.env.example apps/web/.env.local
# впишите туда NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY

pnpm dev
```

Откройте `http://localhost:3000` — должна открыться страница с названием
"Bazar" в шапке (взято из `site_settings`) и списком из 19 категорий
(взяты из `categories`). Если категории не отображаются — проверьте, что
`seed.sql` выполнился без ошибок.

## Шаг 3. Создать проект в Vercel и задеплоить

1. Залейте этот код в репозиторий на **GitHub** (пустой приватный репозиторий, затем `git init`, `git add .`, `git commit`, `git push`).
2. Зайдите на **vercel.com** → войдите через GitHub → **Add New → Project** → выберите репозиторий `ing-market`.
3. В настройках импорта:
   - **Root Directory**: `apps/web` (важно, т.к. это монорепо)
   - **Framework Preset**: Next.js (определится автоматически)
4. В **Environment Variables** добавьте те же три переменные, что и в `.env.local`:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY` (пометьте как secret)
5. **Deploy**. Через 1–2 минуты сайт будет доступен на `*.vercel.app` — это и есть
   "hello world" деплой из Модуля 0. Домен `ing-market.ru` (или выбранный) подключается
   в **Vercel → Domains** после покупки у любого регистратора — это отдельный шаг,
   не блокирует разработку.

## Решения, зафиксированные на этом этапе

- **Карты**: по умолчанию берём **Яндекс.Карты** — лучшее покрытие и детализация
  по Ингушетии среди доступных вариантов, бесплатный лимит достаточен на старте.
  Архитектура (таблицы `lat`/`lng` в `listings` и `cities`) не завязана на
  конкретного провайдера — при необходимости замена на 2ГИС/MapLibre займёт
  только переписывание компонента карты, без изменений в БД. Интеграция —
  в Модуле 5 (Карта).
- **Аккаунты Supabase/Vercel**: создаются вами по инструкции выше (бесплатные
  тарифы полностью достаточны для разработки и раннего запуска). Когда дойдём
  до Модуля 10 (монетизация) и реального трафика — обсудим переход на платные
  тарифы (Supabase Pro, Vercel Pro) под нагрузку.

## Модуль 1: Auth и профиль (email + пароль)

Добавлены: регистрация, вход, выход, восстановление пароля, страница профиля
(имя, телефон, аватарка). Google-вход и подтверждение по SMS — отдельным шагом позже.

### 1. Применить новую миграцию

В Supabase → **SQL Editor** → **New query** → вставить содержимое
`supabase/migrations/0002_auth_profile.sql` → **Run**. Это создаст bucket
`avatars` в Storage и донастроит синхронизацию подтверждения email.

### 2. Настроить Auth в Supabase

**Authentication → URL Configuration**:
- **Site URL**: `http://localhost:3000` (после деплоя на Vercel замените на реальный домен)
- **Redirect URLs**: добавьте `http://localhost:3000/auth/confirm` (и позже — такой же адрес на вашем домене Vercel)

**Authentication → Providers → Email**: убедитесь, что **Confirm email** включён
(это значит, что после регистрации придёт письмо со ссылкой подтверждения).

### 3. Проверить, что всё работает

1. `pnpm dev` (в папке `apps/web`), откройте `http://localhost:3000`.
2. Нажмите **"Регистрация"**, создайте тестовый аккаунт.
3. Проверьте почту — придёт письмо от Supabase со ссылкой. Перейдите по ней.
4. Должны попасть на `/profile` — заполните имя, телефон, загрузите аватарку.
5. Проверьте кнопку **"Выйти"** в личном кабинете, затем вход через **"Войти"**.
6. Проверьте **"Забыли пароль?"** — должно прийти письмо, ссылка ведёт на страницу смены пароля.

Если письма не приходят — проверьте папку "Спам", а также
**Authentication → Logs** в Supabase (там видно, было ли письмо отправлено и с какой ошибкой, если нет).

## Вход через Google

### 1. Применить миграцию

**SQL Editor** → новый запрос → содержимое `supabase/migrations/0003_google_oauth.sql` → **Run**.

### 2. Создать OAuth-ключи в Google Cloud Console

1. Зайдите на **console.cloud.google.com**, войдите обычным Google-аккаунтом.
2. Наверху нажмите на выбор проекта → **"New Project"** → назовите, например, `ing-market` → **Create**.
3. В поиске сверху введите **"OAuth consent screen"**, откройте.
4. Выберите **External** → Create. Заполните:
   - App name: `Bazar`
   - User support email: ваш email
   - Developer contact email: ваш email
   Остальное можно пропустить (Save and Continue на каждом шаге).
5. В поиске введите **"Credentials"**, откройте **"Credentials"** в меню слева.
6. **"+ Create Credentials"** → **"OAuth client ID"**.
7. Application type: **Web application**. Name: любое, например `ing-market-web`.
8. В **"Authorized redirect URIs"** нажмите **"+ Add URI"** и вставьте адрес обратного вызова Supabase — его нужно скопировать из Supabase (см. следующий шаг), обычно он выглядит так:
   `https://nspwqdrvcerfqgjfmszn.supabase.co/auth/v1/callback`
9. **Create**. Появится окно с **Client ID** и **Client Secret** — скопируйте оба, они понадобятся сейчас же.

### 3. Включить Google в Supabase

1. В Supabase: **Authentication → Providers → Google**.
2. Включите тумблер.
3. Вставьте **Client ID** и **Client Secret** из Google Cloud Console.
4. Скопируйте показанный там **Callback URL (for OAuth)** — это и есть тот адрес, который нужно было вставить в Google на шаге 2.8 (если делали в другом порядке — просто вернитесь в Google Cloud Console → Credentials → откройте созданный OAuth client → добавьте туда этот адрес).
5. **Save**.

### 4. Проверить

`pnpm dev`, откройте `/login` или `/register` — должна появиться кнопка **"Войти через Google"**. Нажмите, выберите Google-аккаунт — должно перебросить обратно на сайт, на страницу `/profile`, уже авторизованным.

## Модуль 2: Admin CMS (ядро)

Добавлена админ-панель: категории (создание/редактирование/скрытие/удаление),
настройки сайта (название, описание, контакты, лого), баннеры на главной.
Все изменения видны на сайте сразу, без деплоя.

### 1. Применить миграцию

**SQL Editor** → новый запрос → содержимое `supabase/migrations/0004_admin_cms.sql` → **Run**.
Это создаст таблицы `banners`, `admin_audit_log` и хранилище `site-assets` (для лого/баннеров).

### 2. Назначить себя администратором

По умолчанию все новые пользователи получают только роль "user" — админ-панель
им не видна. Чтобы зайти в неё самому, один раз выполните в **SQL Editor**
(замените email на тот, которым регистрировались на сайте):

```sql
insert into public.user_roles (user_id, role_id)
select p.id, r.id
from public.profiles p
join auth.users u on u.id = p.id
join public.roles r on r.code = 'admin'
where u.email = 'ваш-email@example.com'
on conflict (user_id, role_id) do nothing;
```

### 3. Проверить

1. `pnpm dev`, откройте сайт, войдите под тем самым email.
2. В шапке должна появиться ссылка **"Админка"**.
3. Зайдите в **Категории** — попробуйте добавить/скрыть/удалить категорию, проверьте на главной странице.
4. Зайдите в **Настройки сайта** — смените название, проверьте, что оно поменялось в шапке.
5. Зайдите в **Баннеры** — загрузите картинку, добавьте баннер с местом показа "Верх главной", проверьте, что он появился на главной странице.

## Пивот: ING MARKET → Bazar

Изначальный масштаб ("полный аналог Avito/OLX") пересмотрен в сторону лёгкого,
быстрого прототипа-конкурента berkat.ru — без рекламного мусора у конкурента,
но на порядок стильнее и с нормальным поиском/фильтрами. Само название сайта
поменялось на **Bazar** (внутреннее имя проекта/пакета `ing-market` — не
трогали, это не бренд). Что это меняет по существу:

- **Уже сделанное (Модули 0–5) не переписывалось** — авторизация, RLS,
  Server Actions, загрузка файлов в Storage, объявления, поиск/фильтры,
  карта остаются как есть, это и есть ядро "доски объявлений".
- **Монетизация** — упрощается до простого "поднять в списке" + пометка VIP
  (в схеме `listings` уже есть `is_vip`/`vip_until`), без полноценных
  `shops`/`promocodes`/`wallet_transactions` в UI — это архитектурный задел
  на будущее, не первоочередная задача.
- **Общение с продавцом** — вместо полноценного чата с Realtime: телефон
  продавца кликабельной `tel:`-ссылкой, при желании — простые публичные
  комментарии под объявлением. Быстрее в реализации на порядок, этого
  достаточно для прототипа.
- **SEO-дашборды, отзывы/рейтинг продавца, мультиязычность** — откладываются,
  не приоритет сейчас.
- Список городов/районов исправлен (не хватало городского округа Карабулак и
  ~35 сёл) — см. ниже.

## Контакт с продавцом, избранное, "поднятие в списке"

Вместо полноценного чата: телефон продавца скрыт за кнопкой "Показать
телефон" → кликабельная `tel:`-ссылка, плюс простые публичные комментарии под
объявлением (без вложенности). Добавлено избранное (кнопка на странице
объявления, раздел в личном кабинете) и бесплатное "Поднять в списке" (не
чаще раза в 24 часа — поднимает объявление в выдаче так, будто его только что
опубликовали). В админке при одобрении объявления можно отметить его VIP.

Ручные шаги и известные ограничения — в
[MANUAL_STEPS.md](MANUAL_STEPS.md#контакт-с-продавцом-избранное-поднятие-в-списке).

## Дизайн-система

Первый настоящий дизайн-проход (до этого — голые дефолтные классы Tailwind,
буквально служебная заглушка из Модуля 0). Что сделано:

- Шрифт — **Golos Text** (`next/font/google`, самохостится, без внешних
  запросов) — кириллический шрифт, спроектированный для рус. интерфейсов, а
  не Latin-шрифт "с поддержкой кириллицы" как довесок.
- Палитра — тёплый почти-белый фон + тёплый почти-чёрный текст (одна
  "температура" вместо нейтрального серого по умолчанию) и глубокий зелёный
  акцент — цвет с флага Ингушетии, осознанный выбор, а не дефолт фреймворка.
- Починен реальный баг: `tailwind.config.ts` не определял `muted-foreground`,
  хотя `text-muted-foreground` используется по всему сайту (194 места) — весь
  вторичный текст всё это время рендерился обычным чёрным без иерархии.
  Теперь работает — и это разом улучшило вообще все страницы.
- Иконки категорий — `lucide-react` (line-icons), не эмодзи.
- Глубокий редизайн: шапка/подвал сайта, главная страница (вместо служебного
  текста Модуля 0 — нормальный hero), карточка объявления в каталоге/поиске/
  избранном, страница объявления, панель фильтров, auth-страницы
  (вход/регистрация/восстановление пароля).

**Что ещё не редизайнено** (осталось на дефолтных классах — токены/шрифт уже
подхватились автоматически, но компоновка/отступы не пересматривались):
админ-панель целиком (категории/баннеры/настройки/жалобы/дашборд), "Мои
объявления" и форма подачи объявления, страница профиля. Отдельная задача,
если нужно довести до того же уровня.
